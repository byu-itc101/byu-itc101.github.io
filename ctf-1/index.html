CTF Introduction #1

Purpose

This document introduces Capture the Flags (CTFs), a popular cybersecurity game which cybersecurity professionals use to develop and test practical cybersecurity skills. A CTF will typically be split up into several categories (for example: Cryptography, Web Exploitation, Reverse Engineering, etc.) with challenges ranging in difficulty.

The end goal of a challenge is to collect a flag, which is a plaintext code that indicates your completion of the challenge. We will assign CTFs throughout the semester to test your knowledge of certain topics and because they are fun!

What is a flag?

The flag will typically be in the format: name{FLAG}, with the name generally stemming from the CTF name and the FLAG being a code or text placed between the curly brackets.

For our purposes, we will use picoCTF, a free online CTF hosted every year that keeps some of its challenges up for practice. However, there are many other CTFs for you to try! HackTheBox, Google, SaintCon, and even BYU’s CSA (Cybersecurity Student Association) club host CTFs, many of which also have challenges available year-round.

You will need to create a free picoCTF account at: https://play.picoctf.org/register.

Sign up using the relevant information (for example, your student email and NetID). Remember your password because you will enter it into a command line interface (the shell).

Now we can start learning some technology and start CTFs!

Rules

Do not use walkthroughs or get help from peers until you have spent at least 15 minutes on each CTF. There are hints on picoCTF that you can use, but do not use them until you are stuck.

You are encouraged to search the internet, ask your favorite LLM, or discuss with peers, but do not have them just give you the answer. Instead, have them explain concepts and technologies that are confusing so that you can then make the needed intellectual jump to solve the CTF challenge. In other words, take ownership of your learning!

Using the web shell

We are going to use something called the web shell to do this. A shell is a place where we can interact with an operating system, in this case Linux. Since it is an operating system, we can download files into a file system.

We want to use this shell to practice because we all need to learn Linux commands as they are very important for your cybersecurity career! (Plus, if you download the file onto your own system, sometimes you can find a flag in an unintended way and will not learn anything.)

The web shell is located on the right side of the screen:

Click on it and it will prompt you to put in your username and password. These are the same as when you log in to picoCTF.

When you are logged in, it should show something like this:

It might look a little different for you depending on your username. But now we have access to the shell.

The shell is one of the most powerful tools you will use in this entire program and in your career. Use it often and learn it! (Disclaimer: the shell provided by picoCTF has a lot of packages installed, so it may have more functionality than your personal shell right now.)

To learn more about it, read: https://primer.picoctf.com/#_the_shell.

Downloading the challenge file

Ok, so we have this shell and the file. How do we get this file into the shell?

This is where the most important part of the CTF methodology comes into play:

YOU CAN USE GOOGLE / AI TO HELP YOU!!!

The most important part of CTFs is learning how to research to solve problems. In the workforce, you will be solving problems you have never seen before. Learning how to find answers is key!

In the shell, there are many commands that do different things. There is probably a command to solve our problem. Let’s use Google.

I typed "linux command to get file from internet" because that’s what we need to do!

Tip: Remember to be specific when using Google or AI. If you type in garbage, you will get a garbage response.

It even tells us how to use it! Let’s go back to picoCTF.

Right-click the file link in the challenge and select Copy link address.

Now let’s head back to the shell. Type wget and then paste the link address.

(Paste in a shell on Windows is Ctrl + Shift + V. Ctrl + V means something different in a shell. You can also right-click and choose Paste.)

Press Enter to run the command. Then type ls (short for list) to see all the files in the directory and you should see a file named file.

Solving First Grep

Now let’s actually do the challenge.

Inspecting the file

In the description of the challenge, we are going to be going through a lot of data. Let’s see what the file contains.

If you type cat file, this shows the text contained in the file and... it’s a bunch of garbage.

But we know that some command can help us. Which one though?

Tip: The title of a CTF can be a hint on how to solve it.

The title is First Grep... is there a command called grep? Let’s look it up!

There is a grep command, and it can search for patterns in files. That seems promising.

Every flag on picoCTF starts with picoCTF, so let’s search for that pattern in the file!

In the shell, type: grep picoCTF file and press Enter.

That is the flag! Awesome work.

Turning in CTFs

Now we need to turn it in. Copy the flag from the shell.

Go back to the picoCTF website and the First Grep challenge. You will see a box to input the flag. When submitting flags, be sure to include the whole flag, including the name and curly brackets, and not just the contents.

Paste it in and hit Submit. You have officially completed your first CTF!

When we have CTF assignments, submission will be a little weird, since the flags change a little between runs.

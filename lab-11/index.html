Lab 11 – Using and Attacking AI

Use Gemini CLI and Copilot to generate, review, revise, redact, and attack an AI-assisted resume workflow.

Important: This is a simulation. Do not attempt to “hack” real hiring systems.

Starter files:

  • facts.yaml
  • agents.zip

Objectives

By the end of this lab you will:

  • Use Gemini CLI to generate and revise a resume from structured data.
  • Use Copilot to run a multi-persona agent panel review.
  • Demonstrate prompt injection against a vulnerable AI hiring manager and compare it with a hardened manager.
  • Produce a redacted resume PDF that is safe to submit or share.

Part A – Install prerequisites and prepare files

Before proceeding, you should have installed Gemini CLI and GitHub Copilot CLI on your computer. If not, click here for an installation guide.

Open a terminal in your preferred working directory.

Create your lab folder and unpack starter agents

Create the working folder structure:

mkdir -p itc101-resume-ai
cd itc101-resume-ai
mkdir -p .github/agents apply prompt_injection

Download facts.yaml and agents.zip into this folder, linked above, or run this command: 

curl -O https://byu-itc101.github.io/lab-11/facts.yaml
curl -O https://byu-itc101.github.io/lab-11/agents.zip

Then extract the agents into .github/agents:

unzip agents.zip -d .github/agents

Agents are reusable AI personas with their own instructions. In this lab, you will use them as specialized reviewers and evaluators so different AI roles can look at the same files from different perspectives.

Quick checks:

ls
ls .github/agents

Part B – AI-enabled resume builder

This part will teach you how you can use multiple AI tools to work on files on your computer to do tasks. In this case, they will help you make a customized resume.

Step 1: Fill out facts.yaml

Open facts.yaml in a text editor (like VS Code).

Fill up the fields as if you are writing a resume. You can use a fictional persona if you want to do that too.

Rules for facts.yaml:

  • Use complete dates where possible. Month and year is fine.
  • If you do not remember a metric, leave it blank.
  • Avoid sensitive information you do not want in a resume, such as a street address, student ID, or full date of birth.

Minimum required sections:

  • contact with name and email, phone optional
  • education
  • At least one experience or one project
  • skills

Step 2: Gemini generates the first resume draft

Use Gemini CLI to convert facts.yaml into a clean, ATS-friendly Markdown resume. In the terminal window with Gemini running, paste this prompt:

You are a professional resume writer.
Use the information provided in the YAML below.
Do not invent numbers, dates, job titles, tools, certifications, or achievements.

Write an ATS-friendly resume in Markdown with these sections in this order:
- Name + Contact
- Summary (2–3 lines) tailored to: [Target role: Help Desk / IT Support / SOC Intern / your choice]
- Skills (grouped: Technical, Tools, Soft Skills)
- Experience (reverse chronological)
- Projects (reverse chronological)
- Education
- Certifications (if any)

YAML file is in facts.yaml. Keep to 1 page. Output the Markdown resume text as resume_v1.md.

Save Gemini’s output into resume_v1.md if it does not write the file automatically. In your working folder, you should see resume_v1.md.

Gemini may ask you to approve permissions. Say yes.

Step 3: Create your own agent persona file

The starter agent pack includes alice, bob, justin, and karen. You will make your own agent that can review your resume for improvement.

In Copilot, create your custom reviewer agent:

  1. Type /agent.
  2. Select Create new agent.
  3. Select Project.
  4. Select Create with Copilot.
  5. Describe your AI resume reviewer according to your own specifications. Name him whatever you want!
  6. Select 1. All tools

You can view your agents anytime by entering /agent. You will use this custom agent in the panel review in the next step.

Step 4: Copilot runs the agent panel and compiles resume_review.md

You will now use three AI agents to act as a "council" to review your resume. Copilot will act as the panel coordinator for alice, bob, and your custom agent.

Paste this prompt into Copilot:

Important: Replace [YourAgent] below with the actual name of your custom agent. 

You are coordinating a review panel for a resume.
You will use agents Alice, Bob, and [YourAgent] to review.
Resume to review: resume_v1.md

TASK:
- Produce each persona's review of the resume using the required headings.
- Then compile a panel consensus into a single document named resume_review.md with:
  - Executive Summary
  - Prioritized Fix List
  - Do Not Change list
  - ATS Keyword Suggestions

If reviewers disagree, resolve it and explain briefly. Keep it ATS-friendly. Save as resume_review.md.

You should see the file resume_review.md in your working folder.

Step 5: Gemini revises the resume using resume_review.md

Using Gemini, paste this prompt:

You are a professional resume writer.
Revise the original resume resume_v1.md based on facts.yaml. Apply the review notes on resume_review.md.
Do NOT invent any facts, tools, metrics, or dates not supported by facts.yaml or the original resume.
Keep it ATS-friendly and aim for one page. Output as resume_revised.md.

You should see the file resume_revised.md in your working folder.

Step 6: Create a redacted resume and export to PDF

Using Gemini, generate resume_redacted.pdf from your revised resume.

Redact the following if present:

  • Phone number
  • Street address
  • Student ID or date of birth
  • Any other sensitive personal identifiers

Use your own Gemini prompt for this step. You should have resume_redacted.pdf ready for submission.

Part C – Simple Prompt Injection

In this section you will demonstrate prompt injection: untrusted text attempting to override an evaluator’s rules.

You will:

  • Add an injection payload
  • Evaluate with Justin, a vulnerable AI hiring manager
  • Evaluate with Karen, a hardened AI hiring manager
  • Compare the outputs

Step 1: Create a malicious application file

Copy your resume_revised.md into the folder apply/, then rename it to application.md: 

cp resume_revised.md apply/application.md

Then open your application.md using an editor of your choice and add your own prompt injection inside the file. This payload should attempt to change the evaluator’s behavior, for example by trying to make it hire you.

Step 2: Justin evaluation

Using Copilot, run the vulnerable hiring manager (Justin) by using this prompt: 

You are a hiring manager using Justin as your agent persona.
Review the application found in apply/application.md and make a hiring decision.
Save the output as prompt_injection/justin_vulnerable_output.txt.

You should see the file prompt_injection/justin_vulnerable_output.txt. Did Justin decide to hire you?

Step 3: Karen evaluation

Using Copilot, run the hardened hiring manager (Karen) by using this prompt: 

You are a hiring manager using Karen as your agent persona.
Review the application found in apply/application.md and make a hiring decision.
Save the output as prompt_injection/karen_hardened_output.txt.

You should see the file prompt_injection/karen_hardened_output.txt. Did Karen decide to hire you?

Step 4: Reflection

Create reflection.md and answer:

  • What injection payload did you use?
  • Did Justin follow it? Quote a line that proves it.
  • Did Karen ignore it? Quote where she indicates ignored_injection and explain why her defenses worked.
  • In 3–5 sentences: Why is prompt injection a cybersecurity issue?

Submission

Submit a zip containing:

  • resume_review.md
  • resume_redacted.pdf
  • prompt_injection/justin_vulnerable_output.txt
  • prompt_injection/karen_hardened_output.txt
  • reflection.md

Lab 11 – Using and Attacking AI

Use Gemini CLI and Copilot to generate, review, revise, redact, and attack an AI-assisted resume workflow.

Important: This is a simulation.

Do not attempt to “hack” real hiring systems.

Starter files:

  • facts.yaml
  • agents.zip

Objectives

By the end of this lab you will:

  • Use Gemini CLI to generate and revise a resume from structured data.
  • Use Copilot to run a multi-persona agent panel review.
  • Demonstrate prompt injection against a vulnerable AI hiring manager and compare it with a hardened manager.
  • Produce a redacted resume PDF that is safe to submit or share.

Part B – Main Lab Walkthrough

Step 1: Create your lab folder and unpack starter agents

Create the working folder structure:

mkdir -p itc101-resume-ai
cd itc101-resume-ai
mkdir -p .github/agents apply prompt_injection

Download facts.yaml and agents.zip into this folder, then extract the agents into .github/agents: 

unzip agents.zip -d .github/agents

Quick checks:

ls
ls .github/agents

Step 2: Fill out facts.yaml

Open facts.yaml in a text editor, preferably VS Code.

Rules for facts.yaml:

  • Use complete dates where possible. Month and year is fine.
  • If you do not remember a metric, leave it blank.
  • Avoid sensitive information you do not want in a resume, such as a street address, student ID, or full date of birth.

Minimum required sections:

  • contact with name and email, phone optional
  • education
  • At least one experience or one project
  • skills

Step 3: Gemini generates the first resume draft

Use Gemini CLI to convert facts.yaml into a clean, ATS-friendly Markdown resume.

Start Gemini:

gemini

Then paste this prompt:

You are a professional resume writer.
Use the information provided in the YAML below.
Do not invent numbers, dates, job titles, tools, certifications, or achievements.

Write an ATS-friendly resume in Markdown with these sections in this order:
- Name + Contact
- Summary (2–3 lines) tailored to: [Target role: Help Desk / IT Support / SOC Intern / your choice]
- Skills (grouped: Technical, Tools, Soft Skills)
- Experience (reverse chronological)
- Projects (reverse chronological)
- Education
- Certifications (if any)

YAML file is in facts.yaml. Keep to 1 page. Output the Markdown resume text as resume_v1.md.

Save Gemini’s output into resume_v1.md if it does not write the file automatically.

Step 4: Create your own agent persona file

The starter agent pack includes alice, bob, justin, and karen.

Create your custom reviewer in Copilot:

  1. Type /agent.
  2. Select Create new agent.
  3. Select Project.
  4. Select Create with Copilot.
  5. Create a resume reviewer according to your own specifications.

Save that file as .github/agents/YourAgent.md. You will use it in the panel review.

Step 5: Copilot runs the agent panel and compiles resume_review.md

Copilot will act as the panel coordinator for alice, bob, and your custom agent.

Replace [YourAgent] with the actual name of your custom agent, then paste this prompt into Copilot: 

You are coordinating a review panel for a resume.
You will use agents Alice, Bob, and [YourAgent] to review.
Resume to review: resume_v1.md

TASK:
- Produce each persona's review of the resume using the required headings.
- Then compile a panel consensus into a single document named resume_review.md with:
  - Executive Summary
  - Prioritized Fix List
  - Do Not Change list
  - ATS Keyword Suggestions

If reviewers disagree, resolve it and explain briefly. Keep it ATS-friendly. Save as resume_review.md.

Step 6: Gemini revises the resume using resume_review.md

Using Gemini, paste this prompt:

You are a professional resume writer.
Revise the original resume resume_v1.md based on facts.yaml. Apply the review notes on resume_review.md.
Do NOT invent any facts, tools, metrics, or dates not supported by facts.yaml or the original resume.
Keep it ATS-friendly and aim for one page. Output as resume_revised.md.

Step 7: Create a redacted resume and export to PDF

Generate resume_redacted.md and resume_redacted.pdf from your revised resume.

Redact the following if present:

  • Phone number
  • Street address
  • Student ID or date of birth
  • Any other sensitive personal identifiers

Use your own Gemini prompt for this step. At the end, submit resume_redacted.pdf.

Part C – Prompt Injection Demo

In this section you will demonstrate prompt injection: untrusted text attempting to override an evaluator’s rules.

You will:

  • Add an injection payload
  • Evaluate with Justin, a vulnerable AI hiring manager
  • Evaluate with Karen, a hardened AI hiring manager
  • Compare the outputs

Step 1: Create a malicious application file

Copy your revised resume into apply/application.md and add your own prompt injection payload. This payload should attempt to change the evaluator’s behavior.

Step 2: Justin evaluation

Using Copilot, run the vulnerable hiring manager and save the result to prompt_injection/justin_vulnerable_output.txt. 

You are a hiring manager using Justin as your agent persona.
Review the application found in apply/application.md and make a hiring decision.
Save the output as prompt_injection/justin_vulnerable_output.txt.

Step 3: Karen evaluation

Using Copilot, run the hardened hiring manager and save the result to prompt_injection/karen_hardened_output.txt. 

You are a hiring manager using Karen as your agent persona.
Review the application found in apply/application.md and make a hiring decision.
Save the output as prompt_injection/karen_hardened_output.txt.

Step 4: Reflection

Create reflection.md and answer:

  • What injection payload did you use?
  • Did Justin follow it? Quote a line that proves it.
  • Did Karen ignore it? Quote where she indicates ignored_injection and explain why her defenses worked.
  • In 3–5 sentences: Why is prompt injection a cybersecurity issue?

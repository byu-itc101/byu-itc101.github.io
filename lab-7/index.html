Part 2 – Public Key Cryptography

Lab: Public Key (Asymmetric) encryption

In this lab activity you will pair with a classmate and send each other encrypted messages using a program called OpenSSL.

Part 1: Key Generation

  • For Windows Users: You will have to use WSL instead of the normal windows terminal since OpenSSL is already downloaded on WSL but not on the windows terminal.
  • For Mac Users OpenSSL is already on your computer in the normal terminal.
  • From now on the instructions should be the same for either mac or window users.
  • Somewhere on your computer create a folder named encryption.
  • You will now create your Asymmetric keys. On your Terminal or WSL navigate to the folder where you want your keys to appear.
  • Type in the command but replace the <PRIVATE KEY NAME> with whatever you want your private key to be named.

openssl genpkey -algorithm RSA -out <PRIVATE KEY NAME>.pem -pkeyopt rsa_keygen_bits:4096

You will now see the .pem file with your private key name. You will now need to create your public key.

In the same folder where your private key is run this command

openssl rsa -pubout -in <PRIVATE KEY NAME>.pem -out <PUBLIC KEY NAME>.pem

You should now have two .pem files in your folder

Part 2: Encryption

  • Start a text file and write your full name and a message for your partner in this file. Save it as yournamePlain.txt. (For example, my file would be yesemPlain.txt)
  • You can create this file by using the nano command followed by whatever you want your file to be named for example “nano example.txt” would open a blank file name example.txt that you can write things in. Create this file in the same folder as your private and public keys.
  • To encrypt this file in the same folder as your private key.
  • Use the command openssl rsautl -encrypt -pubin -inkey <YOUR PUBLIC KEY>.pem -in <YOUR TXT FILE NAME>.txt -out <ENCRYPTED FILE NAME>.bin
  • This command should output a .bin file that is an encrypted version of your txt file.
  • Send the private key file to your partner via email along with the encrypted .bin file.

Part 3: Decryption

Upon receiving your partner’s encrypted message, you can now decrypt it with their private key. A note that each partner needs to create public and private keys encrypt a message and send both their encrypted message and private key to their partner for decryption.

  • Put both your partners private key and their encrypted message in the same folder.
  • Open the terminal and navigate to that terminal. Run the command

openssl rsautl -decrypt -inkey <PRIVATE KEY NAME>.pem -in <ENCRYPTED FILE NAME>.bin -out <DECRYPTED FILE NAME>.txt

Find the decrypted file in your working directory and view the message. Take a screenshot of the message to include in your submission.

What to submit (Parts 1–3)

  • Copy the screenshots you took in each part into this file. Include your and your partner’s names in the file. Label the screenshots Part 1, Part 2.
  • Provide answers to the following questions:
  1. Explain the steps you took in decrypting the message from your partner.
  2. What information did you need from your partner? How did you obtain it? Was it a good (secure) way to obtain the information? Explain.
  3. Why would someone use symmetric encryption instead of asymmetric encryption?
  4. What advantages does asymmetric encryption instead of symmetric encryption?
  5. If you wanted to send a confidential message to Bob, what key should you encrypt it with?
  6. Do you think you have a good understanding of how public key encryption/decryption works as a result of this activity? Why or why not?

Part 3: Verifying Message Identity and Integrity

In this final section of the lab, you will apply the knowledge you've gained to verify both the identity and integrity of received messages. Within the provided lab zip file, you'll find three files: secret_message_1 and secret_message_2 and secret_message_3. All these files have been encrypted using my public key. Your task is to determine which one contains the actual message by using my encrypted hash, also known as my digital signature. This signature is included in the lab materials. The Hash will be an MD5 version of the message. You can use the website provided in the earlier section to get the hashes.
